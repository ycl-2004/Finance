# 80 分整改说明

> 日期：2026-05-02
> 状态：已落地到代码与内容结构

## 目标

把 QM Financial Learning Hub 从“内容完整的学习网站”推进到“更可用的会前准备平台”。

本轮优先解决：

- 小白找不到入口。
- 报税和基础财务管理支持不足。
- 预约流程像提交线索，但静态站实际上没有后台接收。
- 信任证明和隐私说明不够明确。
- 首页视觉偏蓝色模板和外部 stock 图。

## 已完成

### 全站搜索入口

Header 接入现有搜索索引，让用户可以直接搜索 TFSA、NOA、报税、房贷、保险、现金流等关键词。

### 新增高频小白场景

新增两个一级场景：

- `tax-season-prep`：报税准备。
- `cashflow-reset`：现金流与债务整理。

每个场景都包含：

- 常见错误。
- 先学什么。
- 资料清单。
- 应该问专业人士什么。
- 相关知识文章。
- 使用边界。

### 新增资料清单

新增：

- 报税准备清单。
- 现金流与债务整理清单。

仍然坚持不上传真实文件、不保存服务器、不要求敏感密码。

### 新增知识文章

新增：

- `knowledge/06-tax-optimization-asset-protection/04-tax-season-prep.md`
- `knowledge/01-comprehensive-financial-planning/03-cashflow-debt-basics.md`

文章只做教育和资料准备，不替代报税、税务、投资、贷款、法律或债务建议。

### 预约闭环诚实化

预约表单改为“保存本机摘要”。确认页提供：

- 电话确认。
- 邮件草稿。
- 返回资料清单。

这样不会让用户误以为后台已经收到正式预约。

### 信任与隐私

关于页增加“如何验证专业人士”的官方入口。

新增 `/privacy`，说明：

- localStorage 本机保存。
- 不上传真实资料。
- 不发送完整 SIN、银行账号、密码或 CVV。
- 本机摘要不等于 QM Financials 已收到资料。

### UI 方向

去掉外部 stock hero 图，改成自定义“Meeting Pack”产品示意。

视觉从蓝色模板感调整为：

- 暖白背景。
- 中性深色文字。
- 克制蓝绿色强调。
- 少量琥珀/米色边界提示。
- 更像顾问准备工具，而不是营销或 AI 工具页。

## 验证重点

本轮需要至少通过：

- `npm run validate:knowledge`
- `npm test`
- `npm run build`

同时新增测试覆盖：

- 报税与现金流场景存在。
- 两个新场景都有资料清单。
- 知识文章总数更新为 42。
